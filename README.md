# DUO Class Notes

An automated pipeline that turns recorded class sessions from the [DUO 6-year Alim course](https://www.duoonline.org/) into a searchable MkDocs study site.

## What it does

1. **Fetch** — Downloads new MP4 recordings from Google Drive
2. **Transcribe** — Converts audio to text using OpenAI Whisper (`large-v3-turbo`)
3. **Generate notes** — Uses Claude (Sonnet) to produce structured Markdown study notes from each transcript
4. **Build** — Syncs notes into `docs/` and regenerates the MkDocs navigation
5. **Publish** — Commits and pushes everything to GitHub (via `run_pipeline.bat`)

## Configuration setup

> **The entire `config/` directory is gitignored** and must be created manually on each machine. Nothing in it is committed to the repo.

### 1. Create the directory structure

```
config/
  classes.yaml
  service_account.json
  templates/
    tfs.md
    hadith.md
    nahw.md
    sarf.md
    fqh.md
```

### 2. classes.yaml

This file defines every class the pipeline knows about. Copy and adapt:

```yaml
classes:
  - subject: hadith          # must be one of: tfs, hadith, nahw, sarf, fqh
    course: "101"            # course number as a string
    name: "Hadith 101"       # human-readable name (used in MkDocs nav)
    teacher: "Teacher Name"
    gdrive_folder_id: "PASTE_GOOGLE_DRIVE_FOLDER_ID_HERE"
    active: true             # true = included in --active-only runs
    semester: "Spring 2026"

  - subject: nahw
    course: "102"
    name: "Nahw 102"
    teacher: "Teacher Name"
    gdrive_folder_id: "PASTE_GOOGLE_DRIVE_FOLDER_ID_HERE"
    active: true
    semester: "Spring 2026"

  # add more classes as needed...

settings:
  whisper_model: "large-v3-turbo"   # or "base", "small", "medium", "large-v3"
  whisper_language: "en"
  llm_model: "claude-sonnet-4-20250514"
```

**Finding the Google Drive folder ID:** open the folder in Google Drive, look at the URL — the ID is the long string after `/folders/`:
```
https://drive.google.com/drive/folders/1ABC123xyz...
                                        ^^^^^^^^^^^^^^ this is the folder ID
```

Set `active: false` for any class you don't want included in routine `--active-only` pipeline runs (e.g. archived semesters).

### 3. Google Drive service account

**Why a service account?** Google's HTTP file download redirects large files through a virus-scan confirmation page, which breaks automated downloads. Authenticating with a service account lets the pipeline call `files().get_media()` via the Drive API directly, bypassing that redirect entirely.

**Folder sharing:** the Drive folders only need to be set to **"Anyone with the link can view"** (standard link sharing). The service account does not need to be explicitly shared on each folder — it can access any publicly linked folder via the API.

If the folders are privately shared (not public), you would instead share each folder directly with the service account's email address (visible in the Cloud Console).

**Setup steps:**

1. Go to [Google Cloud Console](https://console.cloud.google.com/) > **APIs & Services** > **Enable APIs** > search for and enable **Google Drive API**
2. Go to **IAM & Admin** > **Service Accounts** > **Create Service Account** (the name doesn't matter)
3. On the service account's **Keys** tab → **Add Key** → **Create new key** → **JSON**
4. Save the downloaded JSON file as `config/service_account.json`

The service account needs no IAM roles — it only accesses Drive folders via link sharing, not project resources.

### 4. Note generation templates

Each subject needs a Markdown template in `config/templates/{subject}.md`. The template tells Claude what structure to use for the notes. A minimal template looks like:

```markdown
# {course_name} - Study Notes
**Date:** {date}

---

## Session Overview

{overview}

---

## Key Points

{key_points}

---

## Important Vocabulary

| Arabic Term | Meaning |
|---|---|
{vocabulary_table}

---

## Key Takeaways

{takeaways}
```

Customize the sections for each subject. For example, a Hadith template might include sections for the hadith text, chain of narration, and fiqh rulings derived; a Nahw template might include grammar rules and i'rab examples.

### 5. Whisper virtual environment

Transcription runs in a **separate dedicated venv** (not the main project venv) with a GPU-compatible PyTorch build. The path to its Python executable is hardcoded in `scripts/transcribe.py`:

```python
WHISPER_PYTHON = r"C:\Users\moham\whisper-env\Scripts\python.exe"
```

To use transcription on another machine:

1. Create a venv and install Whisper with CUDA-compatible PyTorch:
   ```bash
   python -m venv C:\whisper-env
   C:\whisper-env\Scripts\pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124
   C:\whisper-env\Scripts\pip install openai-whisper
   ```
2. Update the `WHISPER_PYTHON` constant at the top of `scripts/transcribe.py` to point to your venv's `python.exe`.

If you don't have a GPU, change `--device cuda` to `--device cpu` in `transcribe_for_class()` — transcription will still work but will be significantly slower.

### 6. Anthropic API key (optional)

Only needed if using `--backend api`. Store it in Windows Credential Manager — never in a file:

```python
import keyring
keyring.set_password("duo-class-notes", "anthropic_api_key", "sk-ant-...")
```

The default backend (`--backend cli`) uses Claude Code CLI and does not need this.

## Quick start

### Prerequisites

- Python 3.10+
- `ffmpeg` on PATH (required by Whisper)
- A Google Drive service account with access to the class recording folders
- Anthropic API key (for `--backend api`) **or** Claude Code CLI (for `--backend cli`)

### Install dependencies

```bash
pip install -r requirements.txt
```

### Set up config

Follow the [Configuration setup](#configuration-setup) section above to create `config/classes.yaml`, `config/service_account.json`, and the subject templates.

### Run the pipeline

```bash
# Full pipeline for active classes only
python duo.py pipeline --active-only

# Or step by step
python duo.py fetch      --active-only
python duo.py transcribe --active-only
python duo.py notes      --active-only --backend cli --workers 4
python duo.py build

# Check status
python duo.py status
```

### Windows shortcut

Double-click `run_pipeline.bat` — it runs all steps, commits, and pushes to GitHub.

### Preview the site locally

```bash
mkdocs serve
```

## CLI reference

```
python duo.py <command> [options]

Commands:
  fetch        Download new recordings from Google Drive
  transcribe   Transcribe videos using Whisper
  notes        Generate study notes from transcripts
  build        Sync notes to docs/ and update MkDocs
  pipeline     Run all four steps in sequence
  status       Show counts of videos / transcripts / notes per class

Class selection (required for fetch / transcribe / notes / pipeline):
  --all              All classes (active and inactive)
  --active-only      Active classes only
  --subject SUBJECT  Specific subject: tfs | hadith | nahw | sarf | fqh
  --course COURSE    Course number, e.g. 101

Notes options:
  --force            Regenerate notes even if they already exist
  --backend {api,cli}  Use Anthropic API or Claude Code CLI (default: cli)
  --workers N        Parallel workers for note generation (default: 1)
```

## Adding a new class

1. Add an entry to `config/classes.yaml` with the subject, course, semester, teacher, and Google Drive folder ID.
2. Add or reuse a template in `config/templates/{subject}.md`.
3. Run the pipeline with `--subject <subject> --course <course>`.

## Maintenance guide

### Weekly routine

```bash
python duo.py status          # check what's new / behind
run_pipeline.bat              # or: fetch → transcribe → notes → build → push
```

`status` now checks Google Drive live, so the **Remote** column tells you if new recordings are available before you commit to a full run.

### New semester

1. In `config/classes.yaml`, set old semester classes to `active: false`
2. Add new entries for the new semester with the new Google Drive folder IDs
3. Set `active: true` on the new classes
4. The pipeline picks them up automatically on the next `--active-only` run

### Regenerating notes

Notes are skipped if the `.md` file already exists. To force regeneration:

```bash
# Single class
python duo.py notes --subject nahw --course 102 --force

# All active classes
python duo.py notes --active-only --force
```

### When a session has two recordings (P1 / P2)

The pipeline handles this automatically:
- After transcribing, it detects `-p1` / `-p2` files and merges them into a single `{date}.json`
- If you've already manually combined the transcript, the part files are just deleted
- The `already_done` check recognises that if `{date}.json` exists, both part videos are covered — so they won't be re-transcribed on future runs

### Transcript is wrong or missing

```bash
# Re-transcribe a specific video (delete the old JSON first)
del subjects\sarf\102\transcripts\2026-01-30.json
python duo.py transcribe --subject sarf --course 102
```

### Notes are wrong or outdated

```bash
python duo.py notes --subject sarf --course 102 --force
python duo.py build
```

### Status shows "fetch needed" but you don't want to download

The Remote column reflects what's on Drive right now. If recordings are there but you're not ready to process them, just don't run fetch — everything else will continue to show as up to date.

### Checking what changed before committing

```bash
git diff subjects/          # see note content changes
git status                  # see which files are new
```

### Common errors

| Error | Cause | Fix |
|-------|-------|-----|
| `Missing config/service_account.json` | Service account key not on this machine | Copy from secure backup or regenerate in Cloud Console |
| `No credential found for 'anthropic_api_key'` | API key not in Windows Credential Manager | `keyring.set_password("duo-class-notes", "anthropic_api_key", "sk-ant-...")` |
| `CUDA ERROR` during transcription | GPU ran out of VRAM | Close other GPU apps and retry; or switch to `--device cpu` in `transcribe.py` |
| `UnicodeDecodeError` in transcription output | Windows encoding issue | Fixed — `subprocess.run` now uses `encoding="utf-8"` |
| Drive shows `?` in status | Can't reach Google Drive | Check internet / service account JSON is present |
