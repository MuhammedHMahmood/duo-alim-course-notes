# CLAUDE.md — DUO Class Notes

## What this repo is

An automated pipeline that processes recorded lectures from the DUO 6-year Alim course into a searchable MkDocs site. The pipeline: fetches MP4s from Google Drive → transcribes with Whisper → generates structured study notes with Claude → builds the MkDocs site.

## Key files

| File | Purpose |
|------|---------|
| `duo.py` | Main CLI — entry point for all commands |
| `run_pipeline.bat` | Windows batch script — full pipeline + git push |
| `config/classes.yaml` | Class registry (subjects, courses, Google Drive IDs, `active` flag) |
| `config/templates/{subject}.md` | Per-subject prompts used by Claude to generate notes |
| `scripts/common.py` | Shared utilities: config loading, path helpers, credential access via `keyring` |
| `scripts/fetch.py` | Downloads MP4s from Google Drive using service account |
| `scripts/transcribe.py` | Whisper transcription (outputs JSON to `subjects/*/transcripts/`) |
| `scripts/generate_notes.py` | Claude note generation — supports `api` and `cli` backends |
| `scripts/update_mkdocs.py` | Syncs `subjects/*/notes/*.md` → `docs/` and rebuilds `mkdocs.yml` nav |

## Directory conventions

```
subjects/{subject}/{course}/
  videos/       ← .mp4 downloads from Google Drive
  transcripts/  ← .json files from Whisper
  notes/        ← .md study notes generated by Claude (source of truth)

docs/{subject}/{course}/  ← synced copies for MkDocs (never edit directly)
```

Notes live in `subjects/*/notes/`. The `docs/` folder is generated — always edit source notes in `subjects/`.

## Pipeline commands

```bash
python duo.py pipeline --active-only          # full run, active classes only
python duo.py notes --active-only --workers 4 # parallel note generation
python duo.py build                           # sync docs + rebuild nav
python duo.py status                          # counts per class
```

## Whisper venv (hardcoded path)

`scripts/transcribe.py` calls a **separate venv** via a hardcoded path:
```python
WHISPER_PYTHON = r"C:\Users\moham\whisper-env\Scripts\python.exe"
```
This venv has CUDA 12.8 PyTorch for RTX 5080 GPU acceleration. On another machine, update this constant and the `--device cuda` flag. See README for setup steps.

## Config directory (gitignored — must be created manually)

The entire `config/` directory is in `.gitignore`. On a fresh clone, create:

```
config/
  classes.yaml              # class registry — see README "Configuration setup"
  service_account.json      # Google Drive service account JSON key
  templates/{subject}.md    # one template per subject (tfs, hadith, nahw, sarf, fqh)
```

See the **Configuration setup** section of the README for the full `classes.yaml` schema, how to get the Google Drive service account, and template format.

## Credentials (Windows Credential Manager)

API keys are **never stored in files**. They are read at runtime via `keyring`:

```python
import keyring
keyring.set_password("duo-class-notes", "anthropic_api_key", "sk-ant-...")
```

## Active vs inactive classes

`config/classes.yaml` has an `active` boolean per class. `--active-only` skips inactive ones. Toggle `active: true/false` to control what the weekly pipeline processes.

## Note generation backends

- `--backend cli` (default) — calls Claude Code CLI; no API key needed in environment
- `--backend api` — calls Anthropic API directly via `keyring`-stored key

## Subjects

`tfs` (Tafseer), `hadith`, `nahw` (Arabic grammar), `sarf` (Arabic morphology), `fqh` (Fiqh)

## Things to be careful about

- `docs/` is auto-generated. Do not manually edit files there — they get overwritten by `python duo.py build`.
- `mkdocs.yml` nav section is also auto-generated by `update_mkdocs.py`. Only edit the non-nav parts manually.
- The `site/` directory is the built HTML output — do not commit it unless intentional.
- `config/service_account.json` is gitignored and must be set up manually on each machine.
